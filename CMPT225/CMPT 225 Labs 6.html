CMPT 225 Lab Operator Overloading

 

C++ allows a programmer to overload operators. We've seen this (repeatedly) with the assignment operator, but it can be useful with other operators, particularly the comparison operators.

 

What follows is a brief discussion of operator overloading in C++. Operator overloading allows a programmer to define the behaviour of operators like +, or <, or = for a class.

 

Fraction Class

We will use a Fraction class as an example. This Fraction class represents a fraction as two integers, a numerator and a denominator.  Ive provided you with both the header file (immediately below) and the implementation file (at the end of this document). 

 

#pragma once

#include <iostream>

class Fraction
{
public:
       // Constructors and Destructor
       Fraction();
       Fraction(int n, int d);
       ~Fraction();

 

       // PRE:
       // POST: Fraction is simplified, e.g. 3/9, simplifies to 1/3
       void simplify();
      
       // Overloaded comparison operators
       // ==, !=, <, <=, >, >=

private:
       int numerator; //the "top" of the fraction
       int denominator; //the "bottom" of the fraction

       // PRE:
       // POST: Returns the greatest common divisor of a and b
       int gcd(int a, int b)const;

 

       friend std::ostream & operator<<(std::ostream & os, const Fraction & f);

};

 

Preamble - Methods and the Calling Object

Consider this simple method of a Fraction class which is not actually included in my Fraction class.  I'm assuming that this method definition would be in the implementation (.cpp) file of a Fraction class hence the name of the class followed by the scope resolution operator (Fraction::) which tells the compiler that the definition belongs to the Fraction class method.

 

int Fraction::getNumerator()const{
     return numerator;
}

 

Note the reference to numerator in the return statement.  The method might be called in a program like this:

 

Fraction fr1(3,4); //i.e. 3/4
Fraction fr2(5,8); //i.e. 5/8
cout << fr1.getNumerator() << endl; //prints 3

 

Notice that this small example creates two Fractions, fr1 and fr2 and prints the numerator of fr1 (i.e. the value 3). How does the program "know" that it is the numerator of fr1 that is to be printed and not the numerator of fr2?

 

Because this getNumerator() method call belongs to fr1, or to put it another way fr1 is the calling object, or to put it a third (even more direct) way it is fr1 that appears before .getNumerator() so it is fr1's numerator that is referred to by the method. The dot operator (.) is the member operator and so it is fr1's getNumerator method that is called. Of course, if the line had read fr2.getNumerator() then fr2's numerator (5) would be printed.

 

Understanding this is important if you are to successfully implement operator overloading.

 

An Equals Method

Before moving onto operator overloading let's look at the definition of a method (not an operator) that could be used to tell if two Fractions are the same.  Here is the implementation of such a method.

 

bool Fraction::equals(const Fraction & f)const{

       int coVal = numerator * f.denominator;

       int paramVal = f.numerator * denominator;

       return (coVal == paramVal);

}

 

We could test the method like this:

 

Fraction fr1(3,4);
Fraction fr2(5,8);
cout << fr1 << " == " << fr2 << ": " << (fr1.equals(fr2)) << endl;

 

For those of you who have written or used Java classes this use of an equals method should look familiar.  Now let's look at the first line of the method again.

 

int coVal = numerator * f.denominator;

 

Here two different Fractions are referred to, one of which is identified implicitly and the other explicitly.  Let's look at f.denominator first.  It refers to the denominator of a Fraction called f.  This fraction is the parameter of the method, and therefore relates to the argument in the method call.

 

fr1.equals(fr2)

 

In other words, f refers to fr2, because it is fr2 that is being passed to the parameter f.  It is passed as a constant reference, but that makes no difference here. 

 

So, what do numerator and denominator (without f.) refer to?  They follow the same rules discussed in the preamble about calling objects, that is they are not qualified by the name of a Fraction (like f.) so they must be the numerator and the denominator of the calling object.  The calling object is fr1 (remember it is fr1.equals(fr2)).  If equals had been used like this:

 

fr2.equals(fr1)

 

Then the roles would have been reversed, f.numerator and f.denominator would be the parameter fr1's numerator and denominator and numerator and denominator would have belonged to the calling object, fr2.

 

The == Operator

To move from an equals method like the one shown above to the overloaded operator is fairly straightforward.

 

Overloaded Operator Definition

The first thing to do is to write the overloaded operator itself.  This is exactly the same as the equals method I discussed above except for its name.

 

bool Fraction::operator==(const Fraction & f)const{

       int coVal = numerator * f.denominator;

       int paramVal = f.numerator * denominator;

       return (coVal == paramVal);

}

 

And that's it; the only thing that is left is to understand how it is used.

 

Roles in an Operation

Here is the same example as the equals method example, but using the overloaded operator. 

 

Fraction fr1(3,4);
Fraction fr2(5,8);
cout << fr1 << " == " << fr2 << ": " << (fr1==fr2) << endl;

 

If you look at the definition of the operator, it is exactly the same as the equals method in the way in which it refers to two numerators and denominators.

 

int coVal = numerator * f.denominator;

 

But when we use the == operator there is no explicit calling object. We don't write fr1.==(fr2) which is a good thing because it is horribly ugly!  Instead there are two simple rules when you deal with operator overloading. 

 

1.      If the overloaded operator is a method of a class (as it is in this case) the left operand is the calling object and the right operand is the parameter.  So, in the example above, because we wrote fr1 == fr2, fr1 is the calling object while fr2 is the parameter.

2.      If the overloaded operator is a function, and not a method that belongs to a class, the left operand is the first parameter and the right operand is the second parameter.

Fraction Implementation File (.cpp)

The code for the fraction class that you should complete is given below. 

 

Implementation file

#include <iostream>

#include <sstream>

#include <stdexcept>
#include "Fraction.h"

using namespace std;

// Default constructor
Fraction::Fraction(void)
{
       numerator = 0;
       denominator = 1;
}

// Constructor
Fraction::Fraction(int num, int den)
{
       if (den <= 0){

              throw std::invalid_argument("invalid denominator value");

       }

       numerator = num;

       denominator = den;
}

Fraction::~Fraction(void)
{
}

// Returns the greatest common divisor of two integers
int Fraction::gcd(int a, int b)const{
       while (b != 0){
       int temp = b;
       b = a % b;
       a = temp;
       }
       return a;
}

// Simplifies the calling fraction
void Fraction::simplify(){
       int factor = gcd(numerator, denominator);
       numerator = numerator / factor;
       denominator = denominator / factor;
}

 

std::ostream & operator<<( std::ostream & os, const Fraction & f){

       os << f.numerator << "/" << f.denominator;

       return os;

}
