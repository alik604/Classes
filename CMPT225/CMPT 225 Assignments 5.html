Priority Queue Class

 

Implement a PriorityQueue template class to store data of any type.  The priority queue must be implemented with an array based max heap (i.e. priority is given to the item with the highest value).

 

Priority Queue Class Interface

The PriorityQueue class should implement the following public methods:

  constructor creates an empty priority queue of size 4

  constructor (T*, n) creates a priority queue that contains the contents of its first parameter, an array of template variables; the size of this array is given by the second parameter (an integer); the constructor should copy the contents of the array to an underlying array variable of the same size as the array parameter and then convert the underlying array to a heap using the private heapify method.

  copy constructor a constructor that creates a deep copy of its PriorityQueue constant reference parameter

  destructor - de-allocates any dynamic memory associated with the object

  operator= overloads the assignment operator for PriorityQueue objects (deep) copies its PriorityQueue constant reference parameter into the calling object and returns a reference to the calling object after de-allocating any dynamic memory associated with the original contents of the calling object; if the calling object is the same as the parameter the operator should not copy it

  insert inserts its template variable parameter at the appropriate place in the priority queue while maintaining the heap properties; if the priority queue is full the method should double the size of the underlying array, copy the contents of the original array into the new array and then insert the value

  remove removes and returns the value in the priority queue with the highest priority (since this is a max heap this will be the element with the largest value in the priority queue) while maintaining the heap properties; if the deque is empty throws a runtime_error (this error class is defined in the stdexcept library file)

  peek returns the value in the priority queue with the highest priority; if the priority queue is empty the method should throw a runtime_error error

  size returns the number of values currently stored in the priority queue

  maxSize returns the size of the priority queues underlying array 

  empty returns true if the priority queue is empty, returns false otherwise

  sort sorts the contents of its first parameter, an array of template variables, using the heapsort algorithm; the size of this array is given by the second parameter (an integer); the sort should be performed in place on its array parameter, which should not be read into the priority queue's underlying array variable

  contents returns a pointer to the priority queue's underlying array

 

Priority Queue Private Methods and Attributes

The PriorityQueue class should have at least the following private members and methods

  An array of variables of the template type

  An integer to record the maximum size of the underlying array

  An integer to record the number of values stored in the priority queue

  A heapify method the method should convert the contents of its first parameter, an array of template variables into a heap; the size of this array is given by the second parameter (an integer)

Explanation of Class and Methods

Most of the methods described above should be self explanatory. However, some explanation of a few of the methods may be helpful.
 

Heapify Private Method

The heapify private method is intended to convert its array parameter into a heap (in place). This can be useful in the implementation of two public methods. The (relatively) obvious one is the sort method use heapify to convert the sort method's array parameter into a heap. The heapify method can also be used in the <T> array constructor, except that in this case the contents of the constructor's parameter should first be copied into the priority queue's underlying array and the underlying array can then be passed to the heapify method to convert it into a heap.

 

Contents Method

The contents method clearly (I hope) violates the principle of information hiding. It is included in the interface so that we have direct access to your underlying array for testing.

 

Class Design

Including a sort method in the class is not particularly good design as it is not really a coherent part of the data structure. It is included here so that everything I want you to do in the assignment is in one place. You can see that the design is odd since if you want to sort an array, arr, of size n you would first have to declare an empty PriorityQueue object to do it, like this.

 

PriorityQueue pq;

pq.sort(arr, n);
